# Deliverables

This folder is the client-facing package for the MedData Nexus AI security assessment.

Read it in order. The files are numbered to match how a CISO, security manager, or hiring reviewer would naturally consume the work: decision first, proof second, action plan third, reusable operating model last.

## Review Order

| Order | File | Audience | Purpose |
| ---: | --- | --- | --- |
| 1 | `01-executive-summary.md` | CISO / business leader | Gives the scale/no-scale recommendation and the business risk in plain language. |
| 2 | `02-client-findings-report.md` | Security leadership / engineering | Lists findings with rank, impact, framework mapping, owner, remediation, and validation. |
| 3 | `03-remediation-roadmap.md` | Engineering / security owners | Turns findings into immediate, 30-day, 60-day, and 90-day gates. |
| 4 | `04-ai-adoption-operating-model.md` | Security program / AI governance | Shows how the same guardrails apply to RAG, coding assistants, and agentic workflows. |

## What This Proves

The capstone does not stop at technical testing. It turns AI security work into material a client could act on: what failed, why it matters, who owns it, what evidence closes it, and whether the pilot should scale.
