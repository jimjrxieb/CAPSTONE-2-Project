# Readiness Rubrics

These rubrics score whether Capstone 2 is ready to show in an interview, a mentor review, or a client-style walkthrough.

They are not the security findings themselves. They are the quality gates used to decide whether the capstone is clear, defensible, and connected to the AI security role.

## Files

| File | Purpose |
| --- | --- |
| `guidepoint-readiness-rubric.md` | Scores the capstone against the GuidePoint-style AI security engineer signal: architecture, RAG risk, red team scenarios, framework mapping, Eugene evidence, and reviewer explanation. |
| `ai-adoption-readiness-rubric.md` | Scores the reusable client-adoption model: COMPLY, BUILD, BREAK, and PROVE readiness for scaling AI safely. |

## How To Use

Use `guidepoint-readiness-rubric.md` when preparing the capstone for hiring managers and senior AI security reviewers.

Use `ai-adoption-readiness-rubric.md` when explaining how the same method could transfer to a client AI adoption engagement.

The scores should stay honest. A low score is not failure; it marks where the next BUILD/BREAK/PROVE work belongs.
